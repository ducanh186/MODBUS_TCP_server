============================================================
 3A + 5A — Huawei PCS2000HA Full Dynamic Registers + Unit ID
============================================================

Date: 2026-03-26


1. WHAT CHANGED (3A — Full dynamic register blocks)
====================================================

Before: PCS used simplified 0-based registers:
  HR0 = setpoint (int16, scale 0.1 kW, 1 register)
  IR0 = active_power (int16, scale 0.1 kW, 1 register)

After: PCS uses Huawei PCS2000HA address map:
  IR 30000-30088 = Identity + Rating (static, read-only)
  IR 32000-32013 = Running status + Alarms (controller writes bit1=grid-connected)
  IR 32064-32090 = Power readings (controller writes active_power at 32080-32081)
  IR 32463-32468 = Battery cluster on PCS (SOC/SOH placeholders)
  HR 40039-40044 = Control commands (PMS writes setpoint at 40043-40044)

Key register changes:
  OLD: HR0 (1 reg, int16, scale 0.1)    →  NEW: HR 40043-40044 (2 regs, I32, gain=1000)
  OLD: IR0 (1 reg, int16, scale 0.1)    →  NEW: IR 32080-32081 (2 regs, I32, gain=1000)

Legacy 0-based ranges (HR 0-9, IR 0-9) are REMOVED from PCS.
Reading PCS IR0 or HR0 now returns an Illegal Data Address error.


2. WHAT CHANGED (5A — Unit ID = 0)
====================================

All TCP Modbus servers changed: slave_id=1 → slave_id=0
  - PMS  (port 15020): slave_id=0
  - PCS1 (port 15021): slave_id=0
  - PCS2 (port 15022): slave_id=0
  - BMS1 (port 15024): slave_id=0
  - BMS2 (port 15025): slave_id=0
  - Transducer (port 15026): slave_id=0

All inter-device Modbus clients changed: device_id=1 → device_id=0
  - PCS controller → BMS reads: device_id=0
  - PMS controller → PCS writes/reads: device_id=0
  - PMS controller → BMS reads: device_id=0
  - BMS controller → PCS reads: device_id=0
  - Multimeter    → PCS reads: device_id=0
  - External client → PMS reads/writes: device_id=0

NOT changed:
  - Multimeter RTU server: slave_id=10 (RTU convention, separate from TCP)

Note: In Modbus TCP, slave_id=0 is broadcast. pymodbus responds to any
device_id when the server slave_id=0. This is expected — each device
runs on its own TCP port, so port is the real identifier.

4. HOW TO TEST
===============

A) Start plant:
   cd MODBUS_TCP_server/TCP
   python plant.py --no-multimeter

B) Read PCS1 static identity (IR 30000):
   python clients/debug_client.py --port 15021 --slave 0 --area ir --addr 30000 --count 15 --decode raw

C) Read PCS1 running_status (IR 32000):
   python clients/debug_client.py --port 15021 --slave 0 --area ir --addr 32000 --count 1

D) Write demand -500 kW to PMS HR0:
   python clients/debug_client.py --port 15020 --slave 0 --area hr --addr 0 --write 63536
   (63536 = 0xF830 = -500 kW in int16 scale 0.1, two's complement)

E) After 2-3 seconds, read PCS1 active_power (IR 32080-32081, I32):
   python clients/debug_client.py --port 15021 --slave 0 --area ir --addr 32080 --count 2 --decode raw
   Expected: two uint16 registers encoding -250.0 kW as I32 gain=1000
   → raw = -250000 → two's complement 32-bit = 0xFFFC_3CB0 → [0xFFFC, 0x3CB0] = [65532, 15536]

F) Read PMS aggregate (IR 0-4):
   python clients/debug_client.py --port 15020 --slave 0 --area ir --addr 0 --count 5

G) Verify old PCS addresses FAIL:
   python clients/debug_client.py --port 15021 --slave 0 --area ir --addr 0 --count 1
   Expected: Error (Illegal Data Address)

H) Read Transducer frequency (device_id=0):
   python clients/debug_client.py --port 15026 --slave 0 --area ir --addr 0 --count 1
